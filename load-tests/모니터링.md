# 부하 테스트 모니터링 문서

추천 E2E 부하 테스트에서 지연 시간이 증가했을 때 병목 위치를 찾기 위한 문서 묶음이다.

읽는 순서:

1. [Request Flow 아키텍처](./monitoring/01-request-flow.md)
   - 요청 흐름
   - 병목 후보가 생기는 위치
   - 항목별 설정/확인 요약

2. [항목별 설정 방법](./monitoring/02-setup.md)
   - Spring Actuator / Micrometer 설정
   - Prometheus 설정 파일
   - Grafana datasource 설정
   - Docker Compose 설정
   - PostgreSQL slow query log 설정

3. [모니터링 방법](./monitoring/03-how-to-monitor.md)
   - k6 실행 중 무엇을 볼지
   - Grafana에서 어떤 지표를 볼지
   - 터미널에서 Docker/PostgreSQL/Spring 로그를 어떻게 확인할지
   - 관측값을 병목 후보로 해석하는 방법

현재 권장 방식:

```text
Grafana/Prometheus:
Spring Boot Actuator 지표 중심

터미널:
Docker 컨테이너 리소스
PostgreSQL wait_event / slow query log
Spring 로그 thread 이름
```

시간 제약이 있으면 모든 것을 Grafana로 옮기지 않는다. 먼저 Spring 내부 지표만 Grafana로 보고, DB와 컨테이너 상태는 터미널 명령어로 보조 확인한다.
